# 交互模块设计（至少3个，可落地）

> 说明：本包提供3个可实现的交互模块设计，以及对应的数据结构schema（data/interactive_schemas/）。  
> 若要落地为真实交互，请从作者公开仓库或联系作者导出ROI级别数据（正文图多为可视化，不一定包含完整表格）。（证据：PDF第9页 Data and code availability）

---

## 1）交互模块：脑偏离地图（切换两亚型）
1）模块名称与目的：**“脑偏离地图：一键切换两种亚型”**。让用户直观看到两种亚型的偏离方向相反，并理解“偏高/偏低”是相对同龄健康人的统计偏离。  
2）用户交互方式：顶部按钮切换“亚型1/亚型2”；鼠标悬停或点击脑区弹出解释卡（脑区名、偏离方向、偏高/偏低比例、阈值规则）；可切换“发现队列/复现队列”（若数据可得）。  
3）背后展示的数据结构：见 `data/interactive_schemas/brain_deviation_map.schema.json`。（证据：PDF第3–5页 方法与图2）  
4）默认视图与关键文案：默认显示“亚型1 vs 亚型2”的并排视图，并提示：**“红/蓝表示相对同龄健康人的偏离方向，不是好坏评价。”**（证据：PDF第4页 偏离定义）  
5）风险提示：**“本图展示的是群体亚型平均模式，不能用于个人诊断或预测。”**（证据：PDF第3–4页 聚类框架）

---

## 2）交互模块：情绪—认知对照仪表盘
1）模块名称与目的：**“情绪更重还是执行更难？”**把抑郁严重度（HAMD）、执行功能（WCST）与症状网络指标放在同一界面，帮助公众理解两种亚型的“侧重点不同”。  
2）用户交互方式：选择比较组（亚型1/亚型2/健康对照）；点击指标切换（HAMD、WCST各指标、网络密度/强度）；显示“差异方向 + 统计证据”。  
3）背后展示的数据结构：见 `data/interactive_schemas/clinical_cognition_dashboard.schema.json`。（证据：PDF第5–6页 图3）  
4）默认视图与关键文案：默认展示“HAMD总分：亚型2更高”“WCST：亚型1更差”“症状网络：亚型2更密更强”。每一张卡片旁边固定一行解释：**“不同指标回答的是不同问题：情绪痛苦、执行控制、症状互相牵扯。”**（证据：PDF第5–6页 结果与图3）  
5）风险提示：**“量表与测验反映的是平均差异，不能据此给个体贴标签。”**（证据：PDF第9页 Discussion强调诊断与分型差异）

## 3）交互模块：三层证据拼图（影像→细胞/通路→遗传）
1）模块名称与目的：**“三层证据拼图：为什么说两型可能不是同一条路？”**把影像偏离、转录组富集与PRS方向串起来，让用户理解“多维证据一致”与“关联≠因果”。  
2）用户交互方式：左右切换亚型；点击“影像/细胞/遗传”三张卡片逐层展开；在每层末尾显示“我们能说什么/不能说什么”。  
3）背后展示的数据结构：见 `data/interactive_schemas/evidence_chain_puzzle.schema.json`。（证据：PDF第6–8页 图4–5）  
4）默认视图与关键文案：默认先展示一句“影像上方向相反”，再引导用户点击下一层：**“如果影像不同，细胞与遗传线索是否也不同？”**最后用醒目提示收束：**“这些是关联线索；论文未做因果验证与纵向预测。”**（证据：PDF第6–9页 结果与局限性）  
5）风险提示：**“PRS是群体层面的易感度统计，不等于个人必然会患病。”**（证据：PDF第4页 PRS方法；PDF第9页 局限性）
